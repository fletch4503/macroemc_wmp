# MacroGroup Test Task

Django-приложение для учета рабочих мест на производстве.

# Описание задания
Задание подразумевает создание fullstack-приложения с обязательным использованием:
    - **Django + HTMX**: веб-интерфейс для управления учетом рабочих мест с HTMX для обновления интерфейса без JS.
    - **PostgreSQL**: база данных для хранения всех данных приложения.
    - **DaisyUI** как инструмент для структурирования отображения в html-шаблонах приложения Django
    - **Celery**: асинхронные задачи для рассылки уведомлений.
    - **Redis**: кэш и брокер сообщений для Celery. В качестве CELERY_RESULT_BACKEND используется "django-db"
    - **UV**: менеджер зависимостей для Python.
    - **Docker Compose**: оркестрация сервисов (web, db, redis).
## Функционал
  - Необходимо сохранять и отображать информацию о рабочих местах в виде таблицы (списка) на главной странице. 
  - Модель "Рабочие места" (Workplaces) должна состоять из следующий сущностей:
    · Наименование рабочего места(текстовое значение, обязательное, уникальное)
    · Описание рабочего места (текстовое значение)
    · Тип рабочего места - «Место с компьютером», «Место без компьютера»
    · Отдел производства (текстовое значение, обязательное, ForeignKey из таблицы отделов)
    · IP-адрес рабочего места (текстовое значение согласно стандартам IP-адресов)
    · MAC-адрес оборудования рабочего места (текстовое значение согласно стандартам MAC-адресов)
  - Модель "Сотрудник производства" (Staff) должна состоять из следующий сущностей:
    · Имя сотрудника (текстовое значение, обязательное)
    · Фамилия сотрудника (текстовое значение, обязательное)
    · Отдел производства (текстовое значение, обязательное, ForeignKey из таблицы отделов)
    · Email сотрудника (email-поле, обязательное)
    · Внутренний номер телефона сотрудника (5-значное поле, обязательное)
  - Модель "Отдел производства" (Department) должна состоять из следующий сущностей:
    · Название отдела (текстовое значение, обязательное)
    · Описание отдела (текстовое значение, обязательное)
  - Для внесения и отображения данных о рабочих местах на главной странице, помимо Логотипа и Заголовка с описанием 
    производства необходимо разместить таблицу рабочих мест и форму для ввода информации из следующих полей:
    · Поле "Наименование рабочего места"
    · Поле "Описание рабочего места"
    · Поле выбора типа рабочего места из 2-х значений - "Место с компьютером"/"Место без компьютера":
        - Если выбран тип «Место без компьютера», то форма должна работать с 3-мя полями: наименование, описание и отдел. 
        - Если выбран тип «Место с компьютером», то отображается дополнительные поля «IP-адрес» и «MAC-адрес».
    · Поле выбора типа отдела производства (Department).
  - Для Рабочих мест (Workplaces) должны быть доступны все CRUD-операции: 
    · Удаление/Обновление - отдельные кнопки непосредственно в строке основной таблицы
    · Создание - в форме на главной странице, после которой расположена основная таблица. 
      После создания рабочего должна запускаться Celery-задача, эмулирующая отправку сообщения сотрудникам выбранного 
      в предыдущем разделе отдела производства в виде log-сообщения с задержкой в 2 секунды (эмуляция отправки).
      Статус выполнения данной задачи должен отображаться в отдельном разделе главной страницы в online-режиме через
      механизмы HTMX и пропадать по завершении Celery-задачи.
    · Поиск - поле для поиска по наименованию/описанию, расположенной между формой создания и основной таблицей.
      Реализуется также через HTMX-механизмы - при начале ввода текста в окне поиска ниже в таблице должны появляться 
      найденные в модели Workplaces записи, соответствующие 4-м полям - Наименованию, Описанию, Типу и Отделу рабочего 
      места. В поле поиска также должна быть предусмотрена кнопка сброса поиска.
  - Заполнение тестовыми данными через кастомные команды Django с тематикой производства электроники:
    · Для модели "Сотрудник производства" (Staff) - не более 30 записей
    · Для модели "Отдел производства" (Department) - не более 6 записей
    · Для модели "Рабочие места" (Workplaces) - не более 5 готовых записей
  - HTMX для обновления интерфейса без JS
  - Цветное логирование
  - Docker для развертывания
  - Навигационная панель с разделами - Кнопка с логотипом и ссылкой на главную страницу (крайняя левая), Кнопки - 
    Workplaces, Department и Staff (средняя часть панели), переключение темы темная/светлая (крайняя правая)
